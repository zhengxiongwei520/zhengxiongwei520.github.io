使用SASS（SCSS）来实现类样式的复用，或许可以称为通用（基本）样式（工具样式）？就像类设计模式那样，给会经常使用样式包装起来，后续网页需要样式时只需要添加对应类名。

之所以用SCSS，是因为SCSS的语法写嵌套非常舒服，一目了然。

网站并不是看到哪写到哪，可以提前设置一些常用的颜色以及字体。

这2个是SCSS常用的方法，一定要熟练使用。后续还有其他字体大小 边距都是用这种方法实现的，就不贴上来了。

flex布局

flex布局/几乎所有手机都支持/频繁使用，所以flex定义很重要

flex-grow: 1;默认把自己拉伸，比如左边是100px的块，那他会自己填满剩余的空间。

.item {flex: 2333 3222 234px;} 等同于 .item{flex-grow:1; flex-shrink:3222; flex-basis:234px;}

jutify-content是主轴排序方式，flex的方向是水平的，主轴就是水平方向。flex垂直，主轴就是垂直。

align-items是副轴排序方式，flex的方向是水平的，副轴就是垂直方向。flex水平，主轴就是水平。

将scss中的样式变量分开

将scss中运用到的变量拿出来写在_variables.scss中，这个文件被引用的时名字不需要添加下划线。

@import './variables'文件名前面有下划线表示被包含文件，被引用时不用下划线。

把变量分开的原因是有些时候页面需要引入已经设置好的样式，如果不分开，引入的文件就太大了。把变量分开，需要用到时引入变量文件，然后通过map-get()来获取样式

background-color: map-get($colors, "info");

生产环境编译

我手头上的是一个全站项目，分别有三个文件夹，web admin server。如何通过访问server地址可以访问到web admin？那就是把web admin当成静态资源一样在server中暴露出来，前提是吧web admin文件夹加到server中。

step1:改变项目的访问数据库地址

但是当你访问dist的js文件时会发现baseURL是访问的本地连接localhost:3000/admin/api，如果上传到服务器之后肯定是访问不到数据。所以，根据环境不同需要替换baseURL。

step2：将web admin编译到server文件下

step3：server将admin web静态文件暴露出来

购买域名，连接域名

step1:购买域名

aliyu.com购买自己想要的域名。

大陆服务器都需要备案，台湾香港的不需要，但是微信网站小程序不允许非备案网站进行开发。我选择的香港的，因为就是个人做来玩玩。

step2：购买服务器以及配置服务器

aliyun.com购买云服务器ECS。

选择香港服务器最便宜，并且不用备案。突发性能实例 t5 0.05￥/h。

接着选择镜像，其实就是你服务器的操作系统是什么。Ubuntu–18.04 64位

默认专有网络 使用流量计费的10Mbps带宽峰值 安全组默认 使用自定义密码

域名解析

进入阿里云域名控制台，点击解析，添加记录。

域名只是一个别名而已，真正访问的其实是服务器的那个IP地址。解析就是当你访问域名的时候，对应到哪个IP。

Nginx安装和配置

服务器因为是linux系统，所以需要使用nginx来搭建web服务器。

有很多新手朋友在配置了自己的网站后，浏览器输入域名却打不开网站，如果你用的是阿里云的服务器，那么你可能是安全组里面没有放行80端口

登陆阿里云，点击ECS详情信息左边会有实例安全组的设置，点击进去设置

点击操作，接着右上角的添加安全组规则，然后加一条放行80/80端口的记录，授权对象为0.0.0.0/0。接下来可以直接访问域名了，如果现实welcome to nginx就所以你成功了。

nginx反向代理

  1. 安装VSCODE插件REMOTE-SSH，可以像管理本地文件夹一样管理服务器。

安装之后左边图标多了一个电脑图标，点击创建一个SSH TARGERT.

通过ctrl+shift+p打开设置Remote-SSH:Settings，设置Remote.SSH:Show Login Terminal为true，然后就可以连接了。

  • 点击设置配置文件，点击小窗口进入

  • 连接之后打开/etc/nginx/，还需要输入密码。

  • 去nginx配置网站，https://www.digitalocean.com/community/tools/nginx
  • 选择nodejs，进入server，填入自己域名，https不启用，最后下载，反向代理启用注意看端口是否3000，其他都禁用。

  • 把nginxconfig.io文件夹复制过去到/etc/nginx/下，把available里面的文件复制到sites-available文件夹中，把enabled里面的文件复制到sites-enabled文件中。

  • 重载nginx的配置文件，service nginx reload

  • 重启服务器pm2 start server/index.js
