手机端MOBA网站开发总结

发表于 2020-04-18 | 分类于 前端 | 本文总阅读量 次
自己做的一个MOBA游戏移动端网站项目，一个手机端网站里有用的小技巧、设计总体思路。

使用SASS（SCSS）来实现类样式的复用，或许可以称为通用（基本）样式（工具样式）？就像类设计模式那样，给会经常使用样式包装起来，后续网页需要样式时只需要添加对应类名。

之所以用SCSS，是因为SCSS的语法写嵌套非常舒服，一目了然。

npm i -D sass sass-loader安装scss

样式重置，设计网页第一步。要么自己写，要么使用normalize、cssreset，别人写好的常见样式重置。

网站色彩和字体定义

网站并不是看到哪写到哪，可以提前设置一些常用的颜色以及字体。

scss样式可以使用变量！！

这2个是SCSS常用的方法，一定要熟练使用。后续还有其他字体大小 边距都是用这种方法实现的，就不贴上来了。

flex布局

flex布局/几乎所有手机都支持/频繁使用，所以flex定义很重要

flex-grow: 1;默认把自己拉伸，比如左边是100px的块，那他会自己填满剩余的空间。

.item {flex: 2333 3222 234px;} 等同于 .item{flex-grow:1; flex-shrink:3222; flex-basis:234px;}

jutify-content是主轴排序方式，flex的方向是水平的，主轴就是水平方向。flex垂直，主轴就是垂直。

align-items是副轴排序方式，flex的方向是水平的，副轴就是垂直方向。flex水平，主轴就是水平。

常用边距

将scss中的样式变量分开

将scss中运用到的变量拿出来写在_variables.scss中，这个文件被引用的时名字不需要添加下划线。

@import './variables'文件名前面有下划线表示被包含文件，被引用时不用下划线。

把变量分开的原因是有些时候页面需要引入已经设置好的样式，如果不分开，引入的文件就太大了。把变量分开，需要用到时引入变量文件，然后通过map-get()来获取样式

background-color: map-get($colors, "info");

这样就native的类名拥有下划线
