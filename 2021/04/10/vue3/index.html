vue3学习

初识

  1. VUE3支持VUE2的大多数特性,VUE3网址v3.vuejs.org
  2. 性能提升 打包大小减少 初次渲染块 更新快 内存使用减少
  3. composition api 组合api:ref 和 reactive, computed 和watch, 新的生命周期,自定义函数-hooks
  4. 其他特性:teleport 瞬移组件; suspense 异步加载组件; 全局api的修改和优化;
  5. 更好的typescript支持

为什么会有VUE3

vue2的问题
  1. 功能越多,代码越多,复杂组件代码很难维护.

比如,一个列表页面,分页相关的代码分布在不同地方,比如代码就有可能分布在data computed created methods,当前组件代码较少时还比较还改,如果有当前组件有多个跟分页相同复杂度的功能,那么在当前组件要进行后续的修改维护非常困难,一个分页功能修改你需要在跳下跳下的一直查找代码,逻辑并不是在一个函数对象内.

有人说可以用mixin解决.

mixin缺点:1.命名冲突 2.不清楚暴露出来的变量作用 3.重用其他 组件 经常会遇到问题

  1. vue2对typescript支持非常有限

初识响应式 ref computed reactive toRefs

ref用来响应式单个属性,在html中直接调用属性名data就好,但是在ts中调用需要使用data.value

reactive用来响应对象.在html中展示可以obj.keyname来展示,也可以使用toRefs来进行响应式,然后return...refdata,就可以使用属性名调用了.reactive记得使用interface来进行约束,不然不能自动判断类型,会报错

VUE3和VUE2的响应式区别

  1. VUE2响应是使用Object.defineProperty()

缺点:对于对象新增的属性无法响应,对于数组需要拦截原型方法来实现响应式

具体查看 https://cn.vuejs.org/v2/guide/reactivity.html

  1. vue3相应是使用Proxy来实现的 https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Proxy

proxy相比Object.defineProperty()能够感知到所有属性值的变化,并且可以使用更多操作符.

defineComponent是一个函数第一个参数是包裹的组件,他可以给你在写代码时提供很多提示,比如setup methods 生命周期等等.

teleport 就是WAR3 DOTA1 DOTA2 LOL里的传送

Teleport 是一种能够将我们的模板移动到 DOM 中 Vue app 之外的其他位置的技术，非常像游戏里的TP.

为什么需要teleport,如果某个组件被多层嵌套在底层,如果处理组件的定位,样式会很困难,这时候就可以使用teleport.teleport提供了一种干净的方法，允许我们控制在 DOM 中哪个父节点下呈现 HTML，而不必求助于全局状态或将其拆分为两个组件。

App组件中使用,记得要先在app组件中导入modal组件才能使用传送门

Suspense内部组件用来处理异步

  1. 异步请求的困境
  2. Suspense是vue3推出的一个内置的特殊组件
  3. 使用Suspense,需要返回一个promise(完美支持异步组件)

全局API修改

Vue2是这样修改的

Vue2 这样写在一定程度上修改了 Vue 对象的全局状态。

  • 第一，在单元测试中，全局配置非常容易污染全局环境，用户需要在每次 case 之间，保存和恢复配置。有一些 api （vue use vue mixin）甚至没有方法恢复配置，这就让一些插件的测试非常的困难。
  • 第二，在不同的 APP 中，如果想共享一份有不同配置的 vue 对象，也变得非常困难。
