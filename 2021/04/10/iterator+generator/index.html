iterator遍历器以及生成器generator语法及应用

Iterator

遍历器(iterator)就是一个接口,可以访问js各种不同的数据结构,任何数据结构只要部署iterator接口,都可以完成遍历操作.特别是结合for…of循环使用.(目前js一共有数组array 对象object Map Set 4种数据集合).

访问iterator接口

1.for…of循环,只要使用for…of循环会自动寻找iterator接口

for...of循环内部调用的是数据结构的Symbol.iterator方法。for...of循环可以使用的范围包括数组、Set 和 Map 结构、某些类似数组的对象（比如arguments对象、DOM NodeList 对象）、后文的 Generator 对象，以及字符串。
JavaScript原有的for...in循环，只能获得对象的键名，不能直接获取键值。ES6提供for...of循环，允许遍历获得键值。
for..of 循环的等价 for 形式
Set Map结构也可以使用for of访问
ES6的数组、Set、Map都部署了以下三个方法，调用后都返回遍历器对象。
  • entries() 返回一个遍历器对象，用来遍历[键名, 键值]组成的数组。对于数组，键名就是索引值；对于Set，键名与键值相同。Map结构的iterator接口，默认就是调用entries方法。
  • keys() 返回一个遍历器对象，用来遍历所有的键名。
  • values() 返回一个遍历器对象，用来遍历所有的键值。

2.通过[Symbol.iterator]来访问iterator,数据只要是可遍历的,都可以访问这个属性.

generator

1.语法

*+函数声明就是一个generator生成器

跟普通函数一样调用,foo(),只是他不会立马执行(暂停执行,只有调用next时才执行),而是返回一个遍历器对象

然后调用对象的next方法来执行,直到遇到yield或者return停止并返回一个对象{value:’xx’, done:false},value是函数返回值或者yield值,done表示是否已经全部执行完毕.

yield*被称为yield委托,生成器会把这些值给yield出来.所以可以用来yield另一个生成器

2.next方法的参数

yield句本身没有返回值，或者说总是返回undefined。next方法可以带一个参数，该参数就会被当作上一个yield语句的返回值。

3.应用

3.1异步操作的同步化表达

3.2控制流管理

可以用generator来代替 回调套回调 + promise的控制流管理.
