全面解析JavaScript中的this指针

为什么要使用this

​ this是JavaScript的一个很重要的关键字，为什么要使用this？

​ 这段代码可以在不同的执行环境(执行上下文)中重复使用函数 speak() ，不用针对每个对象编写不同版本的函数

​ 如果不使用 this ，需要给函数 speak() 显示的传入一个上下文对象

​ 所以 this 提供了一种更优雅的方法来隐式传递一个对象引用，可以将代码设计得更加简洁、便于复用。

​ 可能这2个例子体现不出this的优势，但是当你的使用模式越来越复杂，显示传递上下文对象会使得代码变得混乱。

this到底指向谁

  1. 指向自身 => 错误

​ 这个例子中，如果this是指向自身，也就是指向foo()函数，那么他输出的应该是0，而不是全局变量中的1，所以指向自身是不对的。

  1. 指向作用域 => 错误

    this在任何情况下都不指向函数的作用域。在 JavaScript 内部，作用域确实和对象相似，作用域内可见的标识符(变量)都是它的属性，但是它只存在于 JavaScript 引擎内部，无法通过 JavaScript 代码直接访问。

  1. 每个函数都有自己的执行环境(执行上下文)，当函数被调用的时候，就会生成执行环境(执行上下文)，他会记录函数在哪被调用（调用栈）、函数的调用方法、传入的参数、this指向等信息。

所以this实际上是在函数被调用时发生的绑定，它的指向完全取决于函数在哪里被调用（函数的调用位置）。

如果想更深入执行环境，去看我的另一篇文章《执行对象、变量对象与作用域链》。

​ 4.寻找调用位置

​ 调用位置就是函数在代码中的被调用位置，而不是声明位置。

绑定规则

1.默认绑定

​ 最常用的函数调用类型，独立函数调用，可以看作无法应用其他规则时的默认规则。

​ 在代码中，bar()直接使用，不带任何修饰直接调用，因此只能进行默认绑定，this指向的全局对象，而(var a=”window”)是全局变量，所以能被访问到。

​ 在严格模式下，不能将全局对象用于默认绑定，this不会指向全局对象window，而是指向undefined。

还有个微妙的细节:

这2个代码区别就在于严格模式开启的地方不同，一个在函数声明内部，一个在函数执行之前。

函数声明内部开启严格模式会“杀掉”this的默认绑定指向全局变量这一规则，所以在后续调用的时候this指向了undefined。

而只是在函数执行前开启严格模式，则不会影响this的默认绑定指向window对象。

2,隐式绑定

这一规则考虑调用位置是否有上下文对象，或者说函数是否被拥有、包含。

​ 当函数调用有上下文对象时（或者说别某个对象“拥有”、“包含”），隐式绑定会把函数中的this指向绑定到这个上下文对象中。

​ 这个例子中，foo()函数调用时拥有上下文对象obj，所以foo()this指向绑定到obj。this.a等同于obj.a。

对象属性引用链只有最后一层在调用位置中有效，所以上述例子this指向绑定到最后一层的obj2对象中。

隐式绑定丢失

这个例子中多添加了一句代码var bar = obj.foo，虽然bar是obj.foo的引用，但是它实际上引用的是foo函数本身，所以bar()其实算一个不带任何修饰的函数调用，因此应用的是默认绑定规则。

​ 传入的参数是obj.foo，应该触发隐式绑定，this指向obj，但是this指向的还是window。为什么呢？

​ 参数传递其实是一种隐式赋值，相当于var fn = obj.foo，所以和上个例子一样，fn虽然是obj.foo的引用，但实际引用的是foo本身，所以还是应用的默认绑定，this指向window对象。

3.显示绑定

​ 显示绑定可以强制改变this的指向。

​ 这里我们就要用到 js 给我们提供的函数 call 和 apply，它们的作用都是改变函数的this指向，第一个参数都是 设置this对象。

​ 两个函数的区别：

  1. call从第二个参数开始所有的参数都是 原函数的参数。
  2. apply只接受两个参数，且第二个参数必须是数组，这个数组代表原函数的参数列表

当第一个参数为null的时候

foo()的this指向被强制绑定到了obj身上，并给foo传入参数。

4.new绑定
什么是new

​ JavaScript中的new使用方法看起来和那些语言相同，通过new创建一个新的对象，但其实new的机制跟那些面向类的语言完全不同。

​ JavaScript中只要用new修饰的函数就是构造函数，它只是函数的构造调用，JavaScript中不存所谓的“构造函数“，只有对函数的“构造调用”。

new会做什么

当使用new来调用函数时（发生构造函数调用时），JavaScript会做以下工作：

​ 1.创建一个全新的对象

​ 2.把新对象的__proto__属性指向函数的prototype属性

​ 3.this指向这个新对象

​ 4.如果函数没有返回其他对象，则返回新对象

优先级

​ new绑定的this > 显示绑定(call apply) > 隐式绑定(obj.foo()) > 默认绑定(独立调用foo())

额外的规则-箭头函数

​ 箭头函数并不是使用function关键字定义的，而是被称为“胖箭头“的操作符 => 定义的。

它和普通函数的区别：

​ 1.书写更方便

​ 2.箭头函数的this绑定无法被修改

​ 3.箭头函数的this值跟外层function的this一致，外层function的this指向谁，箭头函数就指向谁，如果外层不是function就指向window。

bar中的箭头函数外部的function是bar，bar中的this指向为obj，所以箭头函数的this指向也为obj。

baz箭头函数外部没有function，指向window。

小结

​ 找到函数的调用位置，然后用依次使用下面四条规则：

​ 1. 由new调用？this绑定到新创建对象。(var people=new People()绑定到people)

2. 由call或apply调用？绑定到指定的对象。(foo.call(obj)，绑定到obj)
3. 由上下文对象调用？绑定到那个上下文。(obj.foo()，绑定到obj)
4. 默认：在严格模式下绑定到undefined，否则绑定到全局对象。(foo()单独调用，严格模式绑定到undefined，非严格绑定到window对象)
     5. 箭头函数是一个例外：箭头函数的this值跟外层function的this一致，外层function的this指向谁，箭头函数就指向谁，如果外层不是function就指向window。
